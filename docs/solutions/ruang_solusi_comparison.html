Masalah 1 Implementasi Tekstur
LibraryCara kerjaKelebihanKekurangan
ada di AGENTS.md
stb_image.h		 Single-header, drop-in. Load PNG/JPG, bind ke GL_TEXTURE_2D, pakai GL_MODULATE. Nol dependensi tambahan
Sangat ringan (~1100 baris header)		 Tidak ada mipmapping otomatis
Harus generate UV manual
ada di AGENTS.md
SOIL		 Simple OpenGL Image Library — lebih high-level dari stb, punya SOIL_load_OGL_texture(). Satu fungsi langsung load + bind + mipmap Dependensi tambahan, kurang dirawat
Perlu setup build sistem
rekomendasi
stb_image.h + gluBuild2DMipmaps()		 Pakai stb_image untuk load, tapi ganti glTexImage2D dengan gluBuild2DMipmaps() yang sudah built-in di GLU — otomatis generate semua mip level. Nol dependensi baru (GLU sudah ada)
Kualitas tekstur jauh lebih baik
GL_MODULATE tetap bisa dipakai		 Sedikit lebih verbose dari SOIL
alternatif baru
Procedural texture (tanpa file)		 Generate pola checker/noise di CPU sebagai array pixel, upload ke GPU. Tidak perlu load file sama sekali. Tidak ada file aset, portable
Cocok untuk demo/penilaian		 Terlihat kurang realistis
Perlu implementasi noise/pattern sendiri
Rekomendasi: stb_image.h + gluBuild2DMipmaps() adalah sweet spot — tidak tambah dependensi, hasil bagus. Untuk material glossy/rough yang sudah ada, GL_MODULATE cukup (multiply tekstur × material color). Jika mau roughness terasa, gunakan GL_DECAL untuk roughness dan GL_MODULATE untuk glossy — cukup satu baris glTexEnvf() berbeda per mode.
Masalah 2 Implementasi Bayangan (Shadow)
TeknikCara kerjaKelebihanKekurangan
rekomendasi
Projective shadow (stencil)		 Render objek kedua kali via matrix proyeksi ke lantai. Pakai stencil buffer agar tidak overflow keluar area lantai. 100% fixed-function OpenGL 1.x
~30 baris kode tambahan
Cocok dengan directional light yang sudah ada		 Hanya lantai datar
Shadow tidak jatuh ke dinding/objek lain
ada di AGENTS.md
Shadow mapping (glCopyTexImage2D)		 Render depth dari perspektif lampu, simpan ke texture, bandingkan saat render utama. Shadow realistis, jatuh ke semua permukaan Butuh shader untuk perbandingan depth
Tidak praktis di OpenGL 1.x fixed-function
Perlu FBO atau glCopyTexImage2D (error-prone)
alternatif baru
Fake blob shadow		 Render ellipse gelap semi-transparan di bawah tiap objek, scaled berdasarkan posisi Y objek terhadap lantai. Sangat mudah (~10 baris per objek)
Cukup convincing untuk presentasi
Nol kompleksitas matrix		 Tidak mengikuti arah cahaya
Terlihat "game lama", bukan realistis
alternatif baru
Projective + point light approximation		 Projective shadow dengan dua pass: satu untuk LIGHT0 (directional), satu untuk LIGHT1 (point). Shadow di-blend dengan glBlendFunc. Masih fixed-function
Dua shadow terlihat lebih kaya		 Dua kali render objek = perlu optimasi
Blending bisa ada artefak transparansi
Rekomendasi: Projective shadow dengan stencil buffer adalah pilihan terbaik untuk OpenGL 1.x. Shadow mapping membutuhkan shader (GLSL), yang tidak tersedia di stack fixed-function ini. Blob shadow bisa jadi fallback cepat jika waktu terbatas. Formula matrix proyeksi ke lantai (y=0): M = [[lz,0,-lx,0],[0,lz,0,0],[0,0,lz,0],[0,0,0,lz]] dari light position (lx,ly,lz).

Masalah 4 Collision Detection & Boundary
PendekatanCara kerjaKelebihanKekurangan
rekomendasi
AABB (Axis-Aligned Bounding Box)		 Tiap objek punya min/max XZ. Saat placement, cek semua pasangan: if (aMin < bMax && aMax > bMin) → collision. Tolak penempatan. Sangat cepat O(n²) untuk n kecil
Trivial implementasi
Cukup akurat untuk furnitur kotak		 Kurang akurat untuk objek bulat (kursi, meja bundar)
Rotasi objek = AABB membesar, over-reject
ada di AGENTS.md
Bounding cylinder		 Objek bulat pakai silinder bounding: distance(centerA, centerB) < rA+rB. Lebih akurat untuk objek melingkar
Rotation-invariant (tidak terpengaruh rotasi)		 Kurang akurat untuk objek persegi panjang
Perlu dua sistem (AABB + cylinder)
alternatif baru
OBB (Oriented Bounding Box) via SAT		 Seperti AABB tapi ikut rotasi objek. Gunakan Separating Axis Theorem: cek proyeksi di 4 sumbu (2 per objek). Overlap di semua sumbu = collision. Akurat meski objek dirotasi
Tidak over-reject seperti AABB rotasi		 Lebih kompleks implementasi
Mungkin overkill untuk proyek ini
alternatif baru
Grid-based spatial hashing		 Bagi lantai menjadi grid sel (misal 1×1 unit). Tiap sel tahu objek mana yang menempatinya. Collision check hanya pada sel yang sama dan tetangga. O(1) average lookup, skalabel
Juga bisa dipakai untuk snapping ke grid		 Perlu maintain hash map
Kompleksitas lebih tinggi dari AABB sederhana
Rekomendasi: Mulai dengan AABB saja — untuk proyek akademik ini sudah cukup. Untuk objek silindris (kursi, meja bundar), pakai bounding cylinder khusus. Jika perlu rotasi, bungkus AABB dengan padding tambahan 10-15% daripada implementasi OBB penuh. Boundary ruangan cukup dengan clamp posisi ke min/max XZ saat drag.
